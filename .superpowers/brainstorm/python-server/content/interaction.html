合并后的交互模型：两种思路

同一个输入框，AI 自动判断意图 — 但"确认记录"和"对话流"如何切换？

A

纯对话流（聊天式）

所有内容在一列消息流中。说记录→AI 在对话中插入确认卡片；问问题→AI 直接回复。像聊天一样自然，但记录卡片和对话消息混在一起。

B

分离式（记录模式 / 对话模式）

页面有明确分区：上方是「快速记录」区（语音→确认），下方是「问答」区。功能清晰互不干扰，但需要用户知道两个区域的区别。

C

浮动确认卡（混合式）

默认是对话界面。当 AI 识别到记录意图时，从底部弹出一个浮动确认卡片覆盖对话，确认后卡片收起回到对话。兼顾对话历史，又不打断记录流程。
